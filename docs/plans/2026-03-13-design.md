# 闲鱼智能客服设计文档

## 概述

一个基于 Python 的闲鱼自动客服应用，通过 WebSocket 协议实时监听买家消息，结合 LLM 智能生成回复。支持自动回复咨询、智能议价、售后处理。后续计划用 Rust/Tauri 重写为桌面应用。

## 技术栈

- **语言**: Python 3.11+
- **消息通道**: websockets（WebSocket 主力） + Playwright（备用 + Cookie 管理）
- **AI**: OpenAI 兼容 API（默认通义千问 qwen-max）
- **数据存储**: SQLite（对话历史 + 商品缓存）
- **日志**: loguru
- **HTTP 客户端**: httpx（异步请求）

## 双通道消息机制

### WebSocket 通道（主力）

直连 `wss://wss-goofish.dingtalk.com/`（闲鱼底层使用钉钉 IMPaaS 即时通讯平台）。

- 连接时携带 Cookie 换取的 accessToken
- 两阶段注册协议（/reg 注册 + /r/SyncStatus/ackDiff 同步确认）
- 消息格式: JSON-based LWP 协议，内容 Base64 + MessagePack 编码
- 每条服务端消息需回复 ACK
- 心跳保活: 15 秒间隔，5 秒超时
- Token 刷新: 每小时自动刷新，刷新后重连

### Playwright 通道（备用）

通过浏览器自动化操作闲鱼 Web 页面收发消息。

- 作为 WebSocket 通道不可用时的降级方案
- DOM 轮询检测新消息（3-5 秒间隔）
- 操作输入框发送回复

### 通道切换策略

```
正常运行: WebSocket 模式
    │
    ├─ Token 刷新失败 / 连接断开
    │   → 尝试重连（最多 3 次）
    │   → 重连失败 → 切换 Playwright 模式
    │   → Cookie 续期成功 → 切回 WebSocket
    │
    └─ Cookie 完全失效
        → 弹出浏览器通知用户处理
        → 用户过验证后恢复
```

## 认证体系

### Token 管理

- 通过闲鱼 HTTP API 获取 accessToken（MD5 签名认证）
- 刷新间隔 1 小时，失败重试间隔 5 分钟
- 刷新成功后触发 WebSocket 重连
- 失败兜底: 调用 hasLogin 接口刷新 Cookie

### Cookie 管理

- **首次登录**: Playwright 打开登录页，用户扫码，自动提取保存
- **定时续期**: 20~45 分钟随机间隔 + 1~5 秒随机延迟，轮换访问不同页面
- **失效兜底**: 弹出浏览器通知用户手动处理，完成后自动回写恢复
- **持久化**: `data/cookies.json` + `.env` 双写

## 商品信息管理

采用 **API 自动拉取 + YAML 手动补充** 的混合模式。

- **API 拉取**: 根据消息中的 item_id 调用闲鱼商品详情接口，获取标题/描述/价格/SKU，缓存到 SQLite
- **YAML 补充**: `config/products.yaml` 配置 API 拿不到的信息（最低价、自定义卖点、关键词等）
- **合并策略**: YAML 字段优先级高于 API 字段

## 意图路由与 AI 回复

### 三级意图路由

```
买家消息
  │
  ▼
一级：关键词快速匹配（零 LLM 成本）
  商品关键词（优先）→ ProductAgent
  议价关键词       → PriceAgent
  │
  │ 未命中
  ▼
二级：LLM 意图分类（ClassifyAgent）
  → price / product / default / no_reply
  │
  ▼
三级：对应 Agent 生成回复
  prompt.md + 商品信息 + 对话历史 → LLM 生成
```

### Agent 体系

| Agent | 职责 | Temperature |
|-------|------|-------------|
| ClassifyAgent | 意图分类，仅分类不回复 | 0.3 |
| PriceAgent | 智能议价，动态 temperature 随议价次数递增 | 0.3 ~ 0.9 |
| ProductAgent | 商品咨询，结合 API + YAML 信息 | 0.4 |
| AftersaleAgent | 售后处理（**后续迭代**，第一版由 DefaultAgent 兜底） | - |
| DefaultAgent | 通用回复 | 0.7 |

### 安全过滤

所有 LLM 回复检测敏感词（微信/QQ/支付宝/银行卡/线下），命中则替换为平台沟通提醒。

## 对话历史管理

SQLite 存储，三张表:

- **messages**: 按 chat_id 存储对话消息，自动清理超过 100 条的旧消息
- **chat_bargain_counts**: 按 chat_id 追踪议价次数
- **items**: 商品信息缓存

上下文检索按 chat_id 取最近 100 条消息，议价次数作为 system 消息注入。

## 人工接管

卖家发送特定关键词（默认 `#manual`）暂停该会话的自动回复，超时 1 小时自动恢复。人工模式下仍收集消息到对话历史。

## 运行模式

- `python main.py` — 前台运行
- `python main.py --daemon` — 后台守护
- `python main.py --login` — 仅登录/刷新 Cookie

## 配置

### 环境变量

| 变量 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| API_KEY | 是 | - | 通义千问 API Key |
| COOKIES_STR | 是 | - | 闲鱼网页版 Cookie |
| MODEL_NAME | 否 | qwen-max | LLM 模型 |
| TOGGLE_KEYWORDS | 否 | #manual | 人工接管触发词 |
| HEARTBEAT_INTERVAL | 否 | 15 | 心跳间隔（秒） |
| TOKEN_REFRESH_INTERVAL | 否 | 3600 | Token 刷新间隔（秒） |
| MANUAL_MODE_TIMEOUT | 否 | 3600 | 人工接管超时（秒） |
| MESSAGE_EXPIRE_TIME | 否 | 300000 | 消息过期时间（毫秒） |

完整变量列表见 `.env.example`。

### settings.yaml

LLM 配置（base_url、model）和回复配置（max_length、simulate_typing、typing_speed）。兼容 OpenAI API 格式。
